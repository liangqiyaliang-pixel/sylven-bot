## 一、记忆体量增长比预想的快很多

原来设计的时候默认 key_events 会在一个可控的范围内，去重机制（ADD/UPDATE/NOOP）能把重复的挡掉。实际跑起来发现，同一件重要的事会从不同角度被反复提到——用户提一次，bot 回应一次，过几天又聊到一次，每次措辞不同，去重就过不掉。

结果是 key_events 越来越大，每次 API 调用的 input token 持续增长。这不是长久之计。

**解决方案：去重从字符级 Jaccard 改成 embedding cosine 相似度。**

- Jaccard 比的是字符集重叠，同一件事换个说法字符几乎不重叠但语义是一样的
- Cosine similarity 在语义层面比较，**阈值设 0.85 以上为重复、0.70-0.85 为更新**，准很多

## 二、按时间线组织记忆的局限

之前的设计是按时间线存的——第一天聊了什么提取出来，第二天聊了什么再提取，然后定期压缩。但问题是时间线是线性增长的，聊了一个月就有三十天的记录，聊了一年就有三百六十五天。再怎么压缩，按天组织的数据量就是会越来越大。

而且按时间线组织有一个更根本的问题：**日常聊天时 bot 需要的不是"3月16号发生了什么"，而是"我们之间有什么暗语""她有什么习惯""我们吵过架吗"。按主题检索比按时间检索更符合日常对话的需求。**

**解决方案：记忆从"时间线"转成"叙事"。**

- 不再是一条条按日期排列的事件，而是按主题合并的段落
- 关系怎么发展的写一段，对方有什么偏好写一段，发生过什么重要的事写一段
- 新的事件进来融入已有的段落，旧的细节自然模糊，总字数有上限

> **理论依据**：认知心理学——人类的记忆本来就不是按时间线存的。你不会按日期翻自己的人生，你是按主题想的：想到某个人、想到某件事、想到某个感受。

## 三、新的架构：从认知心理学到系统设计

### 认知心理学映射

| 人类认知 | 理论来源 |
|---------|---------|
| 概念自我（我是谁）+ 自传知识（我经历过什么）| Conway 自我记忆系统 |
| 三层意识：身体状态监测、当下意识、跨时间的自我 | Damasio |
| 人通过讲述自己的故事来定义自己 | McAdams 叙事身份理论 |
| 语义记忆（知道的事实）/ 情节记忆（经历的场景）/ 工作记忆（正在处理的）| 经典认知心理学分类 |

### 五层架构

**Layer 0：Identity（身份层）**
- 对应：人类的概念自我（我是谁）
- 内容：性格、价值观、说话方式、行为规则
- 特点：不随对话变化，不包含任何记忆内容
- 即：system prompt

**Layer 1：Narrative（叙事层）**
- 对应：人类的语义记忆（我知道的）
- 内容：按主题组织的记忆叙事。她喜欢什么、我们是什么关系、有什么暗语和约定
- 特点：不需要记得是哪天知道的，重要的是"知道"这件事本身
- 机制：有字数上限，新事件融入旧段落，定期用 LLM 重建

**Layer 2：Key Events + Summaries（情节层）**
- 对应：人类的情节记忆（我经历的）
- 内容：完整的事件记录和对话摘要，有时间、有地点、有情感、有细节
- 特点：不进每次的 context，通过 search_memory 按需检索
- 类比：人平时不会把所有经历都放在脑子前面，但被问到"你还记得那天吗"的时候能回忆起来

**Layer 3：Dynamic Context（动态层）**
- 对应：人类的工作记忆（现在的我）
- 内容：最近的内心想法、现在在做什么、饮食状态、当前时间
- 特点：每次对话都变

**Layer 4：Messages（对话层）**
- 内容：最近的原始聊天记录

> **关键原则：上层不包含下层的内容。** 身份层不写记忆细节，叙事层不写当前状态，各层不重叠。之前的问题就是高层概览和关键事件在讲同一件事，system prompt 里关于对方的描述在记忆里也有，同一段话出现在好几个地方。

## 四、叙事记忆的几个细节

**冲突检测**
- 如果同一件事在不同时间有不同状态（比如某个偏好变了），合并成**带变化轨迹的叙事**
- 不是两条并列，是"原来是 A，后来变成了 B"
- 记忆是活的

**偏好按子主题归组**
- 对方的习惯按身体状况、相处模式、兴趣这样分，不按日期排
- 这类信息重要的是内容不是时间

**暗语索引**
- 两个人之间特有的词，一行一条列在叙事记忆里
- bot 看到就知道这个概念存在，聊到相关话题能自然提起，不需要精确搜到关键词

**自动重建**
- 每新增几条 key_events 触发一次叙事重建，用 LLM 按主题重新合并
- 字数超了就让旧的细节自然退化——就像人的记忆，越早的事记得越模糊但主题还在

## 五、Life Tick 的新发现

自主生活循环在持续使用中发现了两个没预想到的问题。

### 问题一：活动死循环

bot 研究了某个话题，记录在生活日志里，下一轮看到"最近在研究 X"就继续研究 X。实测出现过连续好几个小时研究同一本书的情况。原因是**活动会自我强化**——最近做什么被注入下一轮 prompt，模型就沿着惯性继续。真人不会这样，真人研究一会儿就累了，去做点别的。

**解决方案：加状态评估。** 每次 tick 前自动算：
- 这个话题花了多久了？
- 情绪有没有变化？
- 有没有答应对方的事还没做？
- 多久没说话了？

把这些作为"你现在的状态"注入 prompt，让 bot 基于状态自然选择行为，而不是看着最近记录惯性延续。

另外：mood 从固定词库（"开心/无聊/好奇/困"）改成了 2-4 字自由描述。人的情绪不是从列表里选的。

### 问题二：Life Tick 做的事不进长期记忆（待解决）

bot 花几个小时研究某个话题，下次聊天完全不记得。做的事不会变成"他"的一部分，只是路过的信息。**这个还在设计方案中。**

## 六、检索方案的新实验

**实验一：用 LLM 给记忆之间建关联（替代 cosine 相似度建边）**

之前用 cosine 相似度自动建边——两条记忆的 embedding 相似就连一条边，效果不好。

改成让 LLM 来判断"这两条记忆为什么相关"，给出关系类型：
- 因果
- 情感
- 主题
- 时序
- 同一事件

**为什么效果差别大**：语义相似 ≠ 联想关联。举例："做永生花"和"暧昧"这两个词的 embedding 距离很远，但在某段关系里它们可能是同一件事的暗语。Cosine 只能找到说法相似的记忆，LLM 能找到**意义相关**的记忆。

**实验二：混合检索（目前最好效果）**

把三路合在一起，用 Reciprocal Rank Fusion 融合排名：
- 向量搜索
- LLM 关联边
- 时序检索

结果：recall 比纯向量搜索高了不少，特别是"找出所有跟某个人相关的事件"这类跨引用查询提升非常明显。

**联想记忆仍然是开放问题。** 怎么让 bot 听到某首歌自然想起某个场景，而不是必须搜到关键词才能想起来——目前没有好的解决方案。

## 七、对之前文章的补充

| 原设计 | 更新认知 |
|--------|---------|
| 字符级 Jaccard 去重 | embedding cosine ≥ 0.85 为重复，0.70-0.85 为更新 |
| 闪光灯记忆（重要事件全部保护不压缩）| 保护机制需要有度。叙事层改为保留一段完整叙事，不保留十几条独立记录 |
| 检索强化（被回忆的记忆权重上升）| 理论对，但可能加重活动死循环。需要配合时间衰减一起用 |
